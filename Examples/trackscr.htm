This scenario shows how to use scripts associated with tracks.

Two tracks have scripts associated with them.
The first script shows how to automatically clear a signal
to let the train proceed. The second shows some messages
in the alert window as the state of the track changes.

Description of the activity

Let T1 enter. When T1 reaches the first track of length 500m
it will automatically clear the first signal.
As the signal changes to green, the next track becomes
busy, and so it prints an appropriate message in the alert window.
As T1 reaches the middle track of length 500m, it causes
more messages to be printed, as the train enters and leaves the track.

After the train has exited the territory, try clearing
and closing the first signal. You'll see how the middle track
reacts as it prints the corresponding message.
